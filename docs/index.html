NAME

openosint — Model Context Protocol server and CLI for Open Source Intelligence

SYNOPSIS

openosint [-v] command [args ...]
openosint email address [-t seconds]
openosint username handle [-t seconds]

DESCRIPTION

openosint is a modular OSINT framework that exposes real-world intelligence-gathering tools to large language models via the Anthropic Model Context Protocol (MCP). It also operates as a conventional command-line interface for direct human execution.

The framework is built on a non-blocking asynchronous runtime (Python asyncio) to ensure that long-running subprocess calls do not block the agent reasoning loop. All external binaries are invoked as managed subprocesses with hard timeout enforcement.

openosint does not embed an LLM. It provides the tool surface that an MCP-compatible client (Claude Code, Claude Desktop) connects to and drives autonomously.

ARCHITECTURE

The project is divided into three strictly separated layers:

Layer Location Responsibility
Core tools openosint/tools/ Async wrappers around external OSINT binaries. No I/O, no UI.
MCP server openosint/mcp_server.py Translates core functions into MCP tool schemas. Routes LLM calls.
CLI openosint/cli.py Human-facing interface. Calls core tools directly.

No layer imports from a layer above it. The core tools are stateless and have no knowledge of MCP or argparse.

INSTALLATION

Requires Python 3.10 or later. Installation from source is the recommended method.

$ git clone https://github.com/OpenOSINT/OpenOSINT.git
$ cd OpenOSINT
$ pip install -e .

The following external binaries must be present in PATH:

BinaryPurposeInstall
holehe Email account enumeration pip install holehe
sherlock Username enumeration (300+ platforms) pip install sherlock-project

If a binary is absent, the corresponding tool returns a descriptive error string rather than raising an unhandled exception. The server and CLI remain operational for tools with satisfied dependencies.

CONFIGURATION

Claude Code

Register the MCP server with Claude Code by providing the absolute path to the entry point:

$ claude mcp add openosint python /absolute/path/to/OpenOSINT/openosint/mcp_server.py

Verify registration:

$ claude mcp list

Claude Desktop

Add the following block to ~/Library/Application Support/Claude/claude_desktop_config.json:

{
  "mcpServers": {
    "openosint": {
      "command": "python",
      "args": ["/absolute/path/to/OpenOSINT/openosint/mcp_server.py"]
    }
  }
}

COMMANDS

CommandDescriptionDefault timeout
email ADDRESS Enumerate services registered against an email address via holehe. 120s
username HANDLE Enumerate platforms where a username is registered via sherlock. 180s

Global flags:

FlagDescription
-v, --verboseEnable debug-level logging to stderr.
-t, --timeout NOverride default subprocess timeout (seconds).

EXAMPLES

Enumerate services for an email address with a 60-second limit:

$ openosint email target@example.com -t 60

Search for a username across all supported platforms:

$ openosint username johndoe99

Enable verbose output for debugging a failing scan:

$ openosint -v email target@example.com

Agentic execution via Claude Code after MCP registration:

$ claude
> Investigate target@example.com. If you find an associated
  username, trace it across other platforms.
